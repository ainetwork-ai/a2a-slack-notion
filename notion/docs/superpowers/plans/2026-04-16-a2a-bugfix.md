# Plan: A2A Userflow Bug Fixes

**Source:** `notion/a2a-userflow-report.md` dogfood session (2026-04-16)
**Branch:** main
**Severity summary:** 3 High, 2 Medium

---

## Problem Statement

The dogfood test of the A2A userflow found 5 bugs. All API endpoints work correctly. The bugs are:
- Flow 5 (agent management) is completely inaccessible from the UI (ISSUE-001)
- The editor always shows "Offline" because the WebSocket client uses the wrong port (ISSUE-002)
- The typing indicator badge never renders despite the code existing (ISSUE-003)
- User prompts aren't reaching agents because of message format ordering (ISSUE-004)
- The mentions API silently drops agents when `type` is omitted (ISSUE-005)

---

## Root Cause Analysis

### ISSUE-001 (High) — Sidebar agent items non-interactive

**File:** `notion/apps/web/src/components/sidebar/agent-list.tsx`

Agent items are rendered as `cursor-default` divs with no `onClick`, no hover buttons, no context menu. The API endpoints (`DELETE /agents/:id`, `POST /agents/:id/health`, `GET /agents/:id/skills`) all exist and work.

**Impact:** Flow 5 (Check Status, View Skills, Remove) is entirely inaccessible from the UI.

---

### ISSUE-002 (Medium) — Editor shows "Offline"

**File:** `notion/apps/web/src/components/editor/use-collaboration.ts:38`
**Root cause:** `notion/apps/web` has no `.env.local`. `NEXT_PUBLIC_WS_URL` is undefined at runtime. The fallback hardcode `ws://localhost:3002` is used. But both `.env` files set `HOCUSPOCUS_PORT=3012` — the server runs on 3012, not 3002.

```typescript
// Current (wrong fallback):
const wsUrl = process.env['NEXT_PUBLIC_WS_URL'] ?? 'ws://localhost:3002';

// Fix:
const wsUrl = process.env['NEXT_PUBLIC_WS_URL'] ?? 'ws://localhost:3012';
```

Also create `notion/apps/web/.env.local` with `NEXT_PUBLIC_WS_URL=ws://localhost:3012`.

---

### ISSUE-003 (High) — Typing indicator badge never appears

**File:** `notion/apps/web/src/components/editor/collaborative-editor.tsx`

The badge UI is fully implemented (lines 250–287). The `streamingAgents` state is set on `agent_start` SSE event and cleared on `agent_end`. The bug: React 18 auto-batching groups `setStreamingAgents(add)` and `setStreamingAgents(remove)` together when SSE chunks arrive in a single `reader.read()` result — the intermediate state (badge visible) never commits to the DOM.

Secondary cause: if the SSE path throws any error, the catch block calls `setStreamingAgents([])` and immediately falls back to the non-streaming HTTP endpoint, which inserts content all at once with no badge.

**Fix:** Set an optimistic `invokingAgent` state BEFORE the fetch call. This renders immediately and is visible regardless of SSE batching. The SSE `agent_start` event replaces it with the real streaming badge.

### ISSUE-004 (High) — User prompt not reaching agent

**File:** `notion/apps/api/src/lib/a2a/agent-invoker.ts`

Current message format in both `invokeAgent` and `invokeAgentStream`:
```
[Context: Editing document pageId=X, workspaceId=Y]\n\nUser prompt here
```

A2A agents (and LLMs) treat the first text segment as the primary instruction. The `[Context: ...]` preamble is treated as the full message; the user prompt after `\n\n` is parsed as a continuation (or discarded entirely by simpler agents).

The echo agent used for testing parses `parts[0].text.split('\n\n')[0]` to display the "received prompt", confirming only the context prefix is captured.

Secondary: `agent-mention-handler.ts:75` silently skips invocation if `!prompt` (user typed `@Agent` then Enter with no text). No feedback.

**Fix:**
1. Swap format in `agent-invoker.ts`: prompt first, context second
2. Add empty-prompt fallback in `agent-mention-handler.ts`

```typescript
// New format:
const contextMessage = `${prompt}\n\n[Document context: pageId=${pageId}, workspaceId=${workspaceId}]`;
```

### ISSUE-005 (Medium) — Mentions API excludes agents when `type` omitted

**File:** `notion/apps/api/src/routes/mentions.ts:17`

```typescript
const type = c.req.query('type') ?? 'user';  // defaults to 'user', excludes agents
```

When `type` is absent, the query filters `isAgent: false`. The spec says untyped mention queries should return all workspace members including agents. Current frontend works (it makes two separate calls), but any new client or future refactor would silently miss agents.

**Fix:** Change default to return all workspace members (users + agents) when `type` is omitted.

### Task 3: ISSUE-003 — Typing indicator optimistic state

**File:** `notion/apps/web/src/components/editor/collaborative-editor.tsx`

```typescript
// New state:
const [pendingAgent, setPendingAgent] = useState<{ agentId: string; name: string; color: string } | null>(null);

// In handleAgentInvoke, BEFORE fetch:
setPendingAgent({ agentId: params.agentId, name: '...', color: AGENT_COLORS[...] });

// On agent_start SSE: setPendingAgent(null) (real streamingAgents takes over)
// On agent_end + error: setPendingAgent(null)
```

Render the pending badge in the same badge container:
```tsx
{(streamingAgents.length > 0 || pendingAgent) && (
  <div style={{ display: 'flex', gap: 8 }}>
    {pendingAgent && !streamingAgents.find(a => a.agentId === pendingAgent.agentId) && (
      <AgentBadge agent={pendingAgent} />
    )}
    {streamingAgents.map(agent => <AgentBadge key={agent.agentId} agent={agent} />)}
  </div>
)}
```

The `agentId` for pending state comes from `params.agentId`. The name requires a lookup — add an `agentName` param to `handleAgentInvoke` or look it up from the editor's mention data.

### Task 4: ISSUE-004 — Fix prompt format

**File:** `notion/apps/api/src/lib/a2a/agent-invoker.ts`

Change both `invokeAgent` (line 25) and `invokeAgentStream` (line 72):

```typescript
// invokeAgent — was:
const contextMessage = `[Context: Editing document pageId=${pageId}..., workspaceId=${workspaceId}]\n\n${prompt}`;

// invokeAgent — becomes:
const contextMessage = `${prompt}\n\n[Document context: pageId=${pageId}${blockId ? `, blockId=${blockId}` : ''}, workspaceId=${workspaceId}]`;

// invokeAgentStream — becomes:
const contextMessage = `${prompt}\n\n[Document context: pageId=${pageId}${blockId ? `, blockId=${blockId}` : ''}, workspaceId=${workspaceId}, agentRegistry=${registryUrl}]`;
```

Also fix silent skip in `agent-mention-handler.ts:75`:
```typescript
// was: if (!prompt) continue;
// becomes:
if (!prompt) {
  // Agent mentioned with no prompt — invoke with empty-prompt message
  const fallbackPrompt = 'Please help with this document.';
  extensionOptions.onInvoke?.({
    agentId: mention.id,
    prompt: fallbackPrompt,
    pageId: extensionOptions.getPageId?.() ?? '',
    workspaceId: extensionOptions.getWorkspaceId?.() ?? '',
  });
  continue;
}
```

### Task 5: ISSUE-005 — Mentions API default type

**File:** `notion/apps/api/src/routes/mentions.ts`

```typescript
// was:
const type = c.req.query('type') ?? 'user';

// becomes:
const type = c.req.query('type');  // no default — undefined = all

// Add new branch before the existing type === 'user' check:
if (!type || type === 'all') {
  // Return all workspace members including agents
  const members = await prisma.workspaceMember.findMany({
    where: { workspaceId, user: { name: { contains: q, mode: 'insensitive' } } },
    include: { user: { select: { id: true, name: true, image: true, isAgent: true, a2aUrl: true, agentStatus: true } } },
    take: 10,
  });
  return c.json(members.map(m => ({
    id: m.user.id,
    name: m.user.name,
    avatar: m.user.image ?? undefined,
    ...(m.user.isAgent && { isAgent: true, a2aUrl: m.user.a2aUrl, agentStatus: m.user.agentStatus }),
  })));
}
```
